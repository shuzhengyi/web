# 大模型调用说明文档

## 概述

本项目使用智谱 AI (Zhipu AI) 的 GLM 模型进行 Excel 数据解析。

***

## 1. 使用的模型

* **提供商**: 智谱 AI (Zhipu AI)

* **模型**: `GLM-4.7-Flash`

* **API 地址**: `https://open.bigmodel.cn/api/paas/v4/chat/completions`

* **API 规范**: 兼容 OpenAI Chat Completions 格式

**文件位置**: [src/lib/zhipu-ai.ts](file:///d:/AI/github/traeweb1/web/src/lib/zhipu-ai.ts#L3-L5)

## 2. Prompt 设计思路

### 2.1 系统角色 (System Prompt)

使用**结构化指令**设计，包含：

1. **角色定义**: 明确 AI 的专业角色
2. **任务说明**: 列出需要提取的具体字段
3. **输出格式**: 提供期望的 JSON Schema 模板
4. **注意事项**: 约束输出行为（只返回 JSON、类型要求等）

### 2.2 两个主要 Prompt 模板

#### Prompt 1: `parseExcelWithAI` - Excel 数据解析

* **目的**: 从 Excel 中提取业务数据（单号、收货信息、SKU 列表等）

* **输入**: 转换为 CSV 格式的 Excel 文本（限制前 50 行）

* **输出**: 结构化的 JSON，包含 `commonInfo` 和 `items` 两个部分

#### Prompt 2: `detectExcelFormat` - Excel 格式检测

* **目的**: 自动识别 Excel 表头位置、字段映射关系

* **输入**: 转换为 CSV 格式的 Excel 文本（限制前 30 行）

* **输出**: 包含 `headerRow`、`dataStartRow`、`fieldMapping`、`skipRows` 的 JSON

### 2.3 参数配置

* `temperature: 0.1` - 低温度确保输出稳定、可预测

* `top_p: 0.9` - 核采样参数

## 3. API Key 配置方式

### ⚠️ 当前问题

API Key **硬编码**在源代码中:

```typescript
// src/lib/zhipu-ai.ts 第 3 行
const ZHIPU_API_KEY = '1f2a6045292a4a64b2babc25eb2410ae.AURG2MZxiRi6J3kK';
```

### 建议改进

应改为从环境变量读取，与数据库配置方式一致：

1. 在 `.env` 文件中添加:

   ```env
   ZHIPU_API_KEY=your-api-key-here
   ```

2. 在 `.env.example` 中添加占位符（不提交真实 Key）

3. 修改代码使用 `process.env.ZHIPU_API_KEY`

4. 确保 `.gitignore` 已排除 `.env` 文件
